## 一、产品概览

产品定位：医院建筑节能方案助手，B端工具类产品。帮助医院管理者 / 节能服务公司完成从建筑信息录入到节能方案输出的全流程。

### 核心交互模型：五步线性 Stepper

整个产品是一个严格的线性流程，上一步完成后才能进入下一步：

```
Step 1                Step 2                Step 3                Step 4                Step 5
建筑基本信息     →    节能方案筛选     →    机电系统投资概算     →    能耗分析与节能计算     →    数据分析与辅助决策
（填表）              （技术匹配+选择）      （投资概算+补贴）        （精算+分析）              （多项目对比+决策）
```

**关键约束**：
- 五个步骤由顶部 Stepper 导航条串联，步骤间不可跳转
- 每一步内部可以有子步骤（内部 Stepper 或 Tabs）
- 步骤间数据通过 Zustand store 传递，下一步自动继承上一步的填写结果
- 已完成的步骤允许点击回退查看/修改，但修改后需重新确认后续步骤数据是否过期
- 每个步骤底部有「保存」「上一步」「下一步」按钮

### 背景数据库（预置参考数据层）

三个数据库是提前录入系统的静态参考数据，为四个步骤提供计算依据、技术推荐、政策匹配等支撑。**不是用户操作模块**，用户不可直接编辑。

| 数据库 | 内容 | 支撑哪些步骤 |
|--------|------|-------------|
| 规范标准库 | 医疗约束规范、设计规范、施工验收规范、系统运维规范（国/地/行标） | Step 1（字段校验）、Step 2（技术边界条件）、Step 4（节能计算公式） |
| 系统素材库 | 数字孪生素材、行业参考数值、设备/材料品牌库、技术参数库、预算报价库 | Step 2（技术指标）、Step 3（设备材料选型+报价）、Step 4（能耗参数） |
| 政策绿融库 | 国家/地区能源政策、可再生能源补贴政策、绿色金融政策 | Step 1（市政能源页自动关联）、Step 3（补贴计算）、Step 4（投资回报分析） |

## 三、Step 2 — 节能方案筛选

基于 Step 1 填写的医院信息，从系统素材库中匹配适用的节能技术，供用户筛选确认。

**前置条件**：Step 1 全部保存完成

### 交互形式
内部 3 个子步骤：

```
2-1.技术匹配与筛选 → 2-2.技术详情确认 → 2-3.节能率估算
```

### 子步骤 2-1：技术匹配与筛选

**匹配逻辑**（后台自动，基于 Step 1 数据 + 背景数据库）：
```
Step 1 数据
  → 提取边界条件（医院类型/规模/床位数/气候分区/智能化水平/机电系统信息）
    → 与系统素材库中节能技术的适用条件做向量匹配
      → 按维度权重计算适配度得分
        → 输出推荐等级和排名
```

| 要素 | 说明 |
|------|------|
| ① 边界条件提取 | 从 Step 1 自动提取：医院类型、规模、床位数、气候分区、智能化水平… |
| ② 维度权重赋值 | 关键维度、重要维度、一般维度（系统预设 + 可调） |
| ③ 适配度计算 | 强适配 / 一般适配 / 不适配 |
| ④ 技术推荐 | 强推荐 ≥80分 / 一般推荐 60~79分 / 谨慎推荐 ＜60分 |

**展示方式**：图标视图 / 列表视图 切换

筛选条件：技术名称 / 技术分类 / 推荐等级

**技术卡片字段**：
- 技术名称
- 技术分类（能源高效利用 / 智能控制及优化 / 可再生能源利用）
- 适配度得分（0-100）
- 推荐等级（★★★/★★/★）
- 基准节能率（如 35%~45%）
- 固定投资指标（元/㎡ 或 万元 + 元/㎡）
- 年运行能耗（t Ce/(万㎡·a)）
- 投资回收期
- 核心优势（解决痛点）
- 适用场景

**已有技术数据示例**：

| 技术名称 | 分类 | 得分 | 等级 | 节能率 | 投资指标 | 回收期 |
|----------|------|------|------|--------|----------|--------|
| 相变储热供暖技术 | 能源高效利用 | 100 | ★★★ | 35%~45% | 90元/㎡ | 2~3年 |
| IoT+数字孪生+AI前馈调节 | 智能控制及优化 | 85 | ★★ | 10%~15% | 150万+125元/㎡ | 6~8年 |
| 地源/空气源热泵多能源耦合供热 | 可再生能源利用 | 90 | ★★★ | 30%~40% | 310元/㎡ | 5~7年 |
| 智能照明控制技术 | 智能控制及优化 | 80 | ★★ | 40%~60% | 38元/㎡ | 1~2年 |
| 洁净区域冷热源升级（六管制） | 能源高效利用 | 95 | ★★★ | 25%~35% | 170元/㎡ | 4~6年 |

### 子步骤 2-2：技术卡片详情

点击任意技术卡片后展示完整适配条件：
- 医院类型：综合医院、专科医院等
- 建筑规模：≥60000㎡ 等
- 床位数要求
- 项目属性：新建/改造
- 气候分区：寒冷地区、夏热冬冷地区、严寒地区…
- 能源系统类型：集中式能源系统
- 适用场景：全院机电系统 / 特定科室
- 适用科室

### 子步骤 2-3：节能率估算

勾选要采用的节能技术后，系统自动计算：
- 各技术分项节能率
- 系统级综合节能率（暖通/电气/给排水等系统维度）
- 院级总节能率

支持修改原始方案参数对比查看，支持导出估算结果。

底部按钮：「上一步」「保存」「下一步」（进入 Step 3，带入选中的技术列表）

## 四、Step 3 — 机电系统孪生管理

基于 Step 2 选定的节能技术，构建「楼宇-科室-系统-设备」四级数字孪生模型，并计算投资概算。

**前置条件**：Step 2 已确认技术方案

### 交互形式
内部 3 个子步骤：

```
3-1.基本信息与补贴 → 3-2.单项技术投资计算 → 3-3.投资概算汇总
```

### 子步骤 3-1：基本信息与补贴

自动带入 Step 1 的填写人、所在地。展示关联能源政策与补贴信息（数据来源：政策绿融库）。

**补贴计算方式**（每项技术可单独配置）：
- 按系统初投资：补贴指标 % × 初投资
- 按系统容量：系统容量 × 单位补贴
- 支持省/市多级补贴叠加（如北京市 30% + 区级 15%）

### 子步骤 3-2：单项技术固定投资计算

对 Step 2 选中的每项技术，分别填写投资明细。Tabs 结构：

| Tab | 内容 | 关键字段 |
|-----|------|---------|
| 主要设备表 | 设备名称 / 规格型号 / 品牌 / 数量 / 单价 / 总价 / **是否为主要用能设备** | 标记主要用能设备将用于 Step 4 节能率计算 |
| 主要材料表 | 材料名称 / 规格 / 数量 / 单价 / 总价 | - |
| 安装与调试 | 安装项目 / 费用 | - |
| 运营维护 | 维保费 / 人工运维费 / 管理费（办公、住宿、差旅等） | 三类费用须区分 |

每行为一条明细记录，支持新增/删除行。

### 子步骤 3-3：投资概算汇总

汇总所有选中技术的投资数据，层级展开表格：

**外层**：项目名称 | 所在地 | 总固定投资 | 总初投资 | 总运维费 | 核算状态 | 填写人 | 填写时间 | 操作

**内层**（展开）：技术名称 | 是否补贴 | 补贴额度 | 固定投资 | 初投资 | 运维费 | 补贴金额 | 投资指标 | 操作

底部按钮：「上一步」「保存」「下一步」（进入 Step 4，带入投资概算数据）

## 七、关键业务逻辑

### 7.1 核算状态流转
```
待核算 → 已核算 → 已出报告
```
Step 1 完成 → 待核算；Step 3 完成 → 已核算；Step 4 生成报告 → 已出报告

### 7.2 投资模式
- EMC（合同能源管理）
- BOT（建设-运营-移交）
- 其他

### 7.3 节能技术分类
1. 能源高效利用技术（相变储热、六管制等）
2. 智能控制及优化技术（IoT+AI前馈、智能照明等）
3. 可再生能源利用技术（地源热泵、空气源热泵等）

### 7.4 推荐等级映射
- ★★★ 强推荐：≥80分
- ★★ 一般推荐：60~79分
- ★ 谨慎推荐：＜60分

### 7.5 费用拆解
```
总固定投资 = 初投资（设备+材料）+ 安装与调试
运维费用 = 能源费 + 维保费用 + 运维人工费用 + 管理费用
```

### 7.6 数字孪生层级
```
楼宇 → 科室 → 系统（暖通/电气/给排水/智能化/医疗动力）→ 设备
```

### 7.7 步骤间数据依赖
```
Step 1 输出 → Step 2 输入：医院类型/规模/床位数/气候分区/机电系统/所在地
Step 2 输出 → Step 3 输入：选中的节能技术列表
Step 3 输出 → Step 4 输入：各技术投资概算明细（设备/材料/安装/运维）
Step 4 输出 → 最终报告
```
