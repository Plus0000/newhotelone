## 01 品牌定位

### 1.1 产品定位
面向**医院管理者 / 节能服务公司**的 B 端专业工具，覆盖「建筑信息录入 → 节能方案筛选 → 投资概算 → 能耗计算 → 辅助决策」全流程闭环。

### 1.2 品牌气质
| 关键词 | 设计映射 |
|---|---|
| **专业可信** | 蓝系主色、克制中性灰、数据严谨右对齐、千分位、单位规范 |
| **高效流畅** | 五步线性 Stepper、一站式子流程、系统承担复杂性 |
| **理性精确** | 数字等宽字体、表格 bordered、对比配色不夸张 |
| **现代克制** | 圆润图标、8px 卡片圆角、轻阴影、不滥用动效 |

### 1.3 品牌主色
**`#2B87C9`（科技蓝）** —— 传达医疗级专业与能源科技感。所有"品牌蓝"唯一来源（详见 §03、附录 A）。

### 1.4 视觉基调
Ant Design 5.x 为骨架，在**数据卡片 / 图表 / 报告页**适度跳出默认审美；整体高信息密度、克制留白，服务于专业决策而非营销展示。

## 02 设计原则

承接 `CLAUDE.md` 交互纲领，落为五条可执行原则（**用户体验优先级高于技术整洁与架构优雅**）：

1. **为目标设计，不为功能设计** — 先问"用户要完成什么"，再决定实现。不因技术可行就堆功能。
2. **不要让用户思考** — 交互自解释，无需说明书。默认值合理、推荐先行、模糊需求给最优解。
3. **系统承担复杂性** — 能自动化不手动、能推断不让填、能一步不拆三步（如电价按所在地联动、设备同步 Step 3）。
4. **渐进式展示** — 先核心后细节：Stepper 分步、子流程内嵌、Drawer/Modal 按需展开，不一次性甩全部选项。
5. **反馈引导行动** — 不只报问题，更引导下一步（"数据可能过期 → 请重新确认"，而非干瘪报错）。

> 适用范围：GUI、表单、Stepper、知识库侧边栏、报告、AI 模块——所有用户可触达的层面。

## 04 字体体系

### 4.1 字体族 ✅
```css
/* 中文 / 通用 */
font-family: -apple-system, BlinkMacSystemFont, 'PingFang SC', 'Microsoft YaHei', sans-serif;
/* 数字 / 数据等宽（标准，需全面落地） */
font-family: 'SF Mono', Menlo, monospace;
font-variant-numeric: tabular-nums;
```
> 数字等宽适用：数据表格、七张结果表、KPI 大数、财务指标。目前仅侧边栏落地，须推广（附录 A #4）。

### 4.2 字号阶（px）✅
| 字号 | 角色 |
|---|---|
| 10 / 11 | 角标、极密集说明（克制使用） |
| **12 / 13** | 表格密集内容、标签、次级正文（最高频） |
| **14** | 默认正文 / 表单标签 / 导航（antd 基准） |
| 15 / 16 | 卡片标题、Stepper 标题、小节标题 |
| 18 / 20 / 22 / 24 | 页面/模块大标题 |
| 28–56 | 报告封面、Hero 数值（仅报告页） |

### 4.3 字重 ✅
| 字重 | 角色 |
|---|---|
| 400 | 正文常规 |
| 500 | 中等强调（表头、次级标题） |
| **600** | 标题/强调（主力字重） |
| 700 | 重标题、KPI 大数 |

> **禁止** 800/820/620 非常规字重，统一收敛 700（附录 A #2）。

### 4.4 行高
正文 `1.5`；标题 `1.3–1.4`；密集表格 `1.4`。

## 06 栅格体系

### 6.1 容器 ✅
| 项 | 值 |
|---|---|
| 内容最大宽 | `1200px`，`margin: 0 auto` |
| 左右安全边距 | `24px` |
| 大弹窗最大宽 | `1400px`（`92vw`） |

### 6.2 栅格基线（Ant Design Row/Col）✅🔭
- 基于 antd **24 栅格**；列间距 `gutter={[16, 16]}` 或 `[24, 24]`。
- 表单单列为主；信息卡片用 Row/Col 分 2–4 列。
- 五步 Stepper 用 `display: grid; grid-template-columns: repeat(5, minmax(0,1fr)); gap: 16` ✅。

### 6.3 响应式断点 🔭（规划，沿用 antd 默认）
| 断点 | 宽度 | 策略 |
|---|---|---|
| xs <576 | 移动 | B 端暂不优先，最低保证可读 |
| sm ≥576 / md ≥768 | 平板 | 卡片降为单列、Stepper 可滚动 |
| lg ≥992 / xl ≥1200 | 桌面（**主目标**） | 1200 容器完整布局 |
| xxl ≥1600 | 宽屏 | 容器居中，两侧留白 |

> 当前为桌面优先 B 端工具，移动适配列入后续规划。

## 07 组件体系

### 7.1 按钮 ✅
- 默认＝线框（次要）；`type="primary"`＝实心（主操作）；`primary ghost`＝主色描边（中量，如"计算"）；`type="text"`＝无框（图标/轻操作）。
- 同组排序：**线框在前，实心 primary 在后**；主操作居最右。
- 危险操作：`danger` + `Modal.confirm`。

### 7.2 表格 ✅
- `bordered`，表头底 `#FAFAFA`，描边 `#F0F0F0`/`#E8ECF0`。
- **数字右对齐，说明列（单位/评价）左对齐**，默认左对齐。
- `.map()` 覆写 `onCell` 须 merge 原始对齐样式。
- 隐藏滚动条用 `.hide-scrollbar-table`；结果表配 KPI 指标卡（700 字重 + 数字等宽）。

### 7.3 卡片 ✅
底 `#FFF`、圆角 `8`、描边 `#E8ECF0`、阴影 `0 1px 4px rgba(0,0,0,0.04)`。

### 7.4 Steps / Tabs ✅
- 顶层＝自绘五步卡（§10.2）；子流程＝次级 `Steps`（small）或受控 `Tabs`（`activeKey`+`onChange`）。

### 7.5 表单 ✅
单列、标签左对齐；长表单分组 + 分割线；优先 `Select`/`Radio`；`InputNumber` 限范围给默认值。

### 7.6 反馈 ✅
- `message.success/error/warning`（校验拦截用 warning）。
- 加载：`Spin size="large"` 居中 + `tip`；步骤切换可配 Skeleton。
- 空态：`Empty` 弱灰文案。
- 过期提示：`Modal.warning` 引导重新确认。

### 7.7 圆角 / 阴影 / 层级 ✅
| 圆角 | 6 默认 / **8 卡片** / 4 标签 / 50% 圆形 |
|---|---|
| 阴影 | 基础 `0 1px 4px rgba(0,0,0,0.04)` / 抬升 `0 4px 12px rgba(0,0,0,0.18)` |
| z-index | 固定底栏 10 / 悬浮按钮 1000 / Modal·Drawer antd 默认 / 侧边栏 900 |

## 08 图标体系

### 8.1 双图标库分工 ✅
| 库 | 用途 | 规格 |
|---|---|---|
| `@ant-design/icons` | 通用功能图标（导航、操作、表单、反馈） | 跟随字号，`fontSize: 14` 基准 |
| `lucide-react` | 差异化场景（知识库侧边栏等需圆润审美处） | `size={18}`，`strokeWidth={1.75}`，原生圆角端点 |

### 8.2 选型理由 ✅
antd 图标 path 端点硬直无法 CSS 圆角化；lucide 原生 `stroke-linecap/linejoin: round`，更现代圆润。

### 8.3 语义化（能耗/建筑）✅🔭
步骤与领域图标须语义化：节能/能耗用图表类、建筑用楼宇类。已用：BookOpen（知识库）/ FileText（标准库）/ Database（素材库）/ Landmark（绿融库）。

### 8.4 规范 🔭
- 同一界面图标库与描边粗细统一，不混用粗细。
- 图标色随文字（次要 `#8C8C8C`、强调 `#2B87C9`），不自定义杂色。
- 不在图标下加文字标签（侧边栏已弃此方案）。

## 09 图表体系（ECharts）

> 部分采样自 `CLAUDE.md` + DataAnalysis，多为制定标准 🔭。

### 9.1 配色 ✅🔭
| 角色 | 值 |
|---|---|
| 实际值 | `#2B87C9`（实线/实心） |
| 预测/目标值 | `#8C8C8C`（虚线） |
| 超标预警 | `#FF4D4F` |
| 多系列序 | `#2B87C9 → #52C41A → #FA8C16 → #722ED1` |
| 节能方案 vs 原方案 | 方案＝主色/绿，原方案＝中性灰 |

### 9.2 图表规范 🔭
- **轴**：轴线 `#E8ECF0`，刻度文字 `#8C8C8C` 12px，网格线 `#F0F0F0` 虚线。
- **Tooltip**：白底、圆角 6、阴影 `0 1px 4px rgba(0,0,0,0.04)`、数值千分位 + 单位。
- **图例**：顶部或右侧，12–13px，色块圆角 2。
- **数值**：千分位、单位标右侧；柱/折线数据标签按需，避免拥挤。
- **空/加载态**：图表区用 `Empty` / `Spin`，不留白板。

### 9.3 图表类型约定 🔭
对比＝柱状（技术间/方案间）；趋势＝折线（能耗随时间）；占比＝环形（投资评分总分）；分布＝堆叠柱。

## 11 AI 模块规范 🔭（待实现 / 规划）

> 代码暂无 AI 模块。以下为面向本产品（节能方案 AI 推荐 / 报告生成 / 知识库问答 / 能耗异常洞察）制定的前瞻标准。

### 11.1 AI 视觉识别
- **AI 强调色**：紫 `#722ED1` 作 AI 专属强调（与品牌蓝区分），可配浅底 `#F9F0FF`、渐变 `linear-gradient(135deg,#2B87C9,#722ED1)` 用于 AI 入口/徽标。
- AI 元素统一带"星点/闪烁"语义图标（lucide `Sparkles`），区别于普通功能。

### 11.2 入口与触发
- 入口克制：知识库侧栏「AI 助手」、表单旁「AI 推荐方案」、报告页「AI 生成/润色」——按场景就近放置，不做全局漂浮打扰。
- 渐进式：先给一键建议，再展开可编辑细节（原则 4）。

### 11.3 输出与交互
- **流式输出**：逐字/逐块渲染 + 光标态；可中断（停止生成）。
- **加载态**：思考态用呼吸点动画 + "正在分析能耗数据…"具体文案（原则 5）。
- **消息样式**：AI 气泡浅紫底 `#F9F0FF` / 用户气泡浅灰 `#FAFAFA`，圆角 8，max-width 容器 70%。
- **可操作**：每条 AI 输出附「复制 / 重新生成 / 采纳到表单」。

### 11.4 可信与边界
- **来源标注**：引用规范/政策库时角标可点溯源（衔接知识库三大库）。
- **置信与免责**：方案推荐标"AI 建议，需专业复核"；财务/合规数字必须可追溯到计算引擎，AI 不臆造数值。
- **人审优先**：AI 仅辅助，关键决策（投资/报告结论）保留人工确认。

### 11.5 建议芯片
对话/表单旁提供预设 `Suggestion Chips`（圆角 16 线框胶囊），降低用户输入成本。

## 12 动效规范

> 现有零星动效已采样 ✅，体系为制定 🔭。**克制原则**：动效服务于反馈与引导，不炫技。

### 12.1 时长 Token 🔭
| Token | 值 | 用途 |
|---|---|---|
| fast | `0.15s` | 小元素 hover/点按 |
| **base** | `0.2s` | 默认过渡（侧边栏展开 ✅、卡片 hover） |
| slow | `0.3s` | 抽屉/弹窗、较大位移 |
| 平滑滚动 | `behavior:'smooth'` | 回顶/锚点（报告页 ✅） |

### 12.2 缓动 🔭
默认 `ease`（≈ `cubic-bezier(0.25,0.1,0.25,1)`）；入场 `ease-out`，退场 `ease-in`。

### 12.3 交互动效 ✅🔭
- **Hover 防抖**：侧边栏 open 150ms / close 250ms ✅，避免误触抖动。
- **Hover 反馈**：卡片轻抬升（阴影 0.04→0.18）或主色描边，时长 fast/base。
- **状态切换**：Stepper 步切换淡入；表格行 hover 浅灰底。
- **加载**：Spin + Skeleton，不用闪烁打断。

### 12.4 约束 🔭
- 单次动效 ≤ 0.3s；列表入场错峰 ≤ 50ms/项。
- 尊重 `prefers-reduced-motion`，降级为无位移淡入。
- 禁止大面积持续动画干扰数据阅读。

## 13 暗黑模式规范 🔭（待实现 / 规划）

## 14 Ant Design 映射规范

### 14.1 全局 Token（ConfigProvider）✅
| antd token | 本项目值 |
|---|---|
| `colorPrimary` | `#2B87C9` |
| `borderRadius` | `6` |
| `colorBgLayout` | `#F0F2F5` |
| `colorBgContainer` | `#FFFFFF` |
| `colorBgElevated` | `#FFFFFF` |
| `locale` | `zhCN` |

### 14.2 组件级覆盖 ✅🔭
| 组件 | 约定 |
|---|---|
| `Card` | `colorBgContainer:#FFFFFF`；外层卡片圆角统一 8（覆盖默认 6） |
| `Table` | `bordered`；表头底 `#FAFAFA`（≈ `colorFillAlter`）；数字列右对齐 |
| `Button` | 默认线框/`primary` 实心/`primary ghost` 主色描边；危险 `danger` |
| `Steps` | 子流程用 `size="small"`；顶层五步为自绘非 antd Steps |
| `Modal` | 危险确认 `okType:'danger'`；大弹窗自定义宽高（§10.5） |
| `Message` | success/error/warning 三态约定 |

### 14.3 映射原则 🔭
- 优先用 **ConfigProvider token** 全局统一，少在组件内联硬编码颜色。
- 自定义值（如 `#E8ECF0`、卡片 8 圆角）须在本文件登记后再用，禁止散落魔法值。
- antd 5.x 为基线；`SKILL .md` 虽列 4.x 文档，API 以项目实际 antd 5.x 版本为准。
